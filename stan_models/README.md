# Stan Models

`stan_models/` contains the Bayesian scoring models used by the Stan workflow.
Each model estimates team attack and defense strengths, a shared scoring level
`eta`, scale parameters for team strengths, and optionally a Dixon-Coles
low-score correction `rho`.

## Naming Convention

Model names combine three choices:

| Prefix / Part | Meaning |
| --- | --- |
| `n_` | Normal priors for attack and defense strengths. |
| `st_` | Student-t priors for attack and defense strengths. |
| `poisson` | Independent Poisson scoring model. |
| `dc` | Dixon-Coles correction for low-scoring results. |
| `ranking` | Attack priors include `prior_strength * beta_prior` from FIFA ranking data. |
| `noranking` | Attack priors are centered at zero. |

For example, `st_dc_ranking.stan` uses Student-t strength priors, the
Dixon-Coles correction, and FIFA ranking information.

## Shared Data Contract

All `.stan` files expect:

- `N`: number of matches;
- `T`: number of teams;
- `team_i`, `team_j`: 1-based team IDs for home and away teams;
- `y_i`, `y_j`: observed goals;
- `game_weight`: per-match likelihood weights;
- `prior_strength`: normalized ranking prior vector, even for `noranking`
  models where it is not used by the attack prior.

## Outputs Used by Python

The Python scripts read posterior variables such as:

- `attack`
- `defense`
- `eta`
- `rho` for Dixon-Coles models

Saved posterior draws are written to `data/outputs/models/*.npz` by
`model_sel/validate.py` and `simulations/train_2026.py`.

## Generated Binaries

Compiled Stan executables may appear in this folder. They are platform-specific
build artifacts and can be regenerated by `cmdstanpy` from the `.stan` files.
